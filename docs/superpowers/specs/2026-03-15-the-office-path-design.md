# The Office Path - 打工之道 设计文档

## 一、项目概述

一个基于 AI Agent 工作流的打工模拟器。玩家从应届毕业生入职第一天开始，在规则与 AI 共同编织的开放世界中，努力工作升职加薪。成为公司高管或创立自己的公司后进入第二阶段，从个人成长转向公司运营。

**核心卖点：涌现式剧情** — 多个 AI Agent 独立决策，叠加产生不可预测的剧情走向，连开发者也不知道下一秒会发生什么。

## 二、核心交互形式

**混合模式：**

- **文字叙事** — 主体交互，AI 生成有画面感的故事文字，像读小说一样体验打工人生
- **结构化仪表盘** — 属性面板、关系网、手机面板等可视化组件，让玩家掌握关键信息
- **选择卡片** — 在关键节点弹出行动选项，玩家做出策略决策

## 三、技术栈

| 层面 | 选型 | 理由 |
|------|------|------|
| 框架 | Next.js（全栈） | 前后端一体，AI 代码生成成熟，部署方便 |
| 语言 | TypeScript | 类型安全，AI 编码出错率低 |
| 状态管理 | Zustand | 轻量，适合游戏状态 |
| AI 后端 | 多模型可切换架构 | 抽象 AI 接口层，支持 OpenAI / Claude / DeepSeek 等灵活替换 |
| 数据持久化 | 先 LocalStorage，后续加云存档 | MVP 优先验证核心玩法 |
| UI 风格 | 像素/复古风，中文为主 | 贴合打工人文化调性 |

## 四、系统架构

### 分层架构

整个游戏分为五层，各层职责清晰：

**1. 前端层（玩家看到的）**

三大区域：故事区（文字叙事）、仪表盘（属性/关系/手机）、行动区（选择卡片）。像素复古风。

**2. 游戏引擎层（规则裁判）**

负责硬规则校验。所有数值变更（金钱、属性、职级）必须经过这里。不讲故事，只管算数和判定对错。

**3. Agent 编排层（调度导演）**

每次玩家做出行动，编排层决定调用哪些 Agent、按什么顺序、如何合并它们的输出、如何解决冲突。

**4. Agent 层（四个编剧）**

四个专职 AI Agent，各自独立决策（详见第九节）。

**5. AI 适配层（翻译官）**

统一接口，底层可切换不同 AI 模型。不同 Agent 可以使用不同模型（叙事用强模型，简单判定用便宜模型）。

## 五、游戏阶段

### 第一阶段：打工人成长

玩家从应届毕业生开始，目标是升职加薪，最终成为高管或选择创业。

**职级体系（参照中国互联网大厂）：**

| 职级 | 称呼 | 月薪参考 | 说明 |
|------|------|---------|------|
| L1 | 实习生 | 3K | 起点，还没转正 |
| L2 | 初级员工 | 8K | 转正，干活的螺丝钉 |
| L3 | 中级员工 | 15K | 能独立负责模块 |
| L4 | 高级员工 | 25K | 团队骨干，开始带新人 |
| **L5** | **资深/主管** | **35K** | **分水岭：选技术线还是管理线** |
| L6(技术) | 技术专家 | 50K | 技术线：攻坚难题 |
| L6(管理) | 经理 | 50K | 管理线：带一个小团队 |
| L7(技术) | 高级专家 | 80K | 技术线：架构级影响力 |
| L7(管理) | 总监 | 80K | 管理线：管一个部门 |
| L8 | VP / CTO | 150K+ | 高管层，进入权力中心 |

**晋升条件（硬规则，完整表）：**

| 晋升 | 条件 |
|------|------|
| L1→L2（转正） | 在职≥1季度，完成实习考核（关键期事件） |
| L2→L3 | 专业能力≥30，完成≥2个项目，在职≥2季度 |
| L3→L4 | 专业能力≥45，沟通能力≥25，绩效≥1次A或2次B+，在职≥3季度 |
| L4→L5 | 专业能力≥60，绩效连续2季度A/B+，直属领导好感≥50，在职≥4季度 |
| L5→L6(技术) | 专业能力≥70，声望≥30，完成≥1个重大项目 |
| L5→L6(管理) | 沟通能力≥50，管理能力≥40，人脉≥30，直属领导好感≥60 |
| L6→L7 | 对应路径核心能力≥80，声望≥50，绩效连续3季度A/B+，在职L6≥4季度 |
| L7→L8 | 声望≥80，两项以上能力≥80，需要"机遇"事件触发（前任离职、公司扩张等） |

到达 L8 或在 L6+ 时选择辞职创业 → 进入第二阶段。

### 第二阶段：公司运营

不再用个人职级，改为公司发展阶段：

| 阶段 | 员工数 | 资金门槛 | 核心挑战 | 升级条件 |
|------|-------|---------|---------|---------|
| 车库创业 | 1-3人 | 启动资金≥10万 | 活下去，找到产品方向 | 季度营收≥10万，连续2季度盈利 |
| 小型团队 | 4-10人 | 现金流≥30万 | 招对人、分好工 | 季度营收≥50万，团队满意度≥60 |
| A轮公司 | 11-50人 | 融资≥500万 | 管理体系搭建 | 季度营收≥200万，获得A轮融资 |
| 成长期 | 51-200人 | 现金流≥500万 | 部门协作、文化建设 | 季度营收≥1000万，年增长率≥50% |
| 上市冲刺 | 200-500人 | 融资≥5000万 | 合规、财务透明化 | 连续4季度盈利，通过上市审核（关键期事件） |
| 上市公司 | 500+人 | — | 终局 | — |

#### 第二阶段从第一阶段继承的状态

- 金钱（个人存款转为创业启动资金）
- 人脉（决定初始可招募的人才质量和融资渠道）
- 声望（影响初始品牌知名度和客户信任）
- 健康、心情（创业者也是人，身体还是会垮）
- 所有 NPC 关系保留（前同事可能被你挖来、投资你、或成为竞争对手）

#### 第二阶段体力行动（季度制，10点体力）

| 行动 | 体力消耗 | 效果 |
|------|---------|------|
| 打磨产品 | 2点 | 提升产品质量，影响营收 |
| 招聘面试 | 2点 | 招募新员工（质量受人脉和声望影响） |
| 融资路演 | 3点 | 接触投资人，争取融资（通过画饼通） |
| 团队管理 | 2点 | 提升团队满意度和效率 |
| 商务拓展 | 2点 | 开拓客户和合作伙伴 |
| 市场营销 | 2点 | 提升公司知名度 |
| 休息调整 | 1点 | 恢复健康和心情 |

#### 第二阶段公司属性

| 属性 | 说明 |
|------|------|
| 季度营收 | 公司收入，受产品质量、市场和行业环境影响 |
| 季度支出 | 员工工资 + 房租 + 运营成本 |
| 现金流 | 营收 - 支出，连续2季度为负则面临倒闭危机 |
| 公司估值 | 由营收、增长率、行业热度综合计算，影响融资 |
| 员工数量 | 影响产能和支出 |
| 团队满意度 | 0-100，太低导致核心员工离职 |
| 产品质量 | 0-100，影响营收和口碑 |

#### 第二阶段失败条件

- 现金流连续2季度为负且无法融资 → 公司倒闭 → 游戏结束（可选择重新打工回到第一阶段）
- 团队满意度降至0 → 核心团队集体离职 → 公司瘫痪
- 健康降至0 → 创始人病倒，公司群龙无首

## 六、规则系统（分层设计）

### 硬规则（游戏引擎强制执行）

所有数值计算、晋升条件、经济系统由代码严格控制，AI 不能违反。

### 软规则（AI 自由发挥）

人际关系变化、办公室政治、随机生活事件、行业风云、剧情走向由 AI Agent 自由发挥。

**核心原则：硬规则保证游戏"公平可玩"，软规则保证游戏"有趣意外"。**

## 七、时间与体力系统

### 双节奏时间系统

**平淡期（季度推进）：**

每季度玩家获得 **10 点体力**，分配到各种行动中：

| 行动 | 体力消耗 | 效果 |
|------|---------|------|
| 埋头工作 | 2点 | 提升绩效，小幅提升专业能力 |
| 学习充电 | 2点 | 提升某项能力值（可选方向） |
| 社交应酬 | 2点 | 提升某个NPC好感度或拓展人脉 |
| 向上管理 | 2点 | 提升直属领导好感，但同事可能反感 |
| 摸鱼休息 | 1点 | 恢复心情值 |
| 搞副业 | 3点 | 赚外快，但有被发现的风险 |
| 跳槽面试 | 3点 | 打探外部机会，可能拿到offer |

每季度结算时：

- AI 根据玩家行动分配 + 随机事件，生成季度叙事总结
- 更新数值（工资、能力、关系等）
- 可能触发晋升评审、裁员、跳槽等关键事件

**关键期（按天推进）：**

进入重大事件时（入职第一周、晋升答辩、公司危机、融资谈判等），切换为每天行动，体力按天计算（每天3点），事件结束后恢复季度模式。

**关键期触发与退出规则：**

| 类型 | 触发条件 | 持续天数 | 退出条件 |
|------|---------|---------|---------|
| 入职第一周 | 游戏开始 | 5天 | 固定5天结束 |
| 晋升答辩 | 满足晋升条件时自动触发 | 3天 | 答辩结果出炉（成功或失败） |
| 公司危机（裁员/收购） | 事件Agent生成 | 5-10天 | 危机解决或玩家做出关键选择 |
| 项目冲刺 | 事件Agent生成 | 3-7天 | 项目交付或失败 |
| 跳槽谈判 | 玩家收到offer后选择谈判 | 3天 | 接受/拒绝offer |
| 创业启动 | 玩家选择创业 | 7天 | 完成公司注册和初始团队搭建 |
| 融资谈判（第二阶段） | 玩家发起融资路演 | 5天 | 融资成功或失败 |
| 上市审核（第二阶段） | 满足上市冲刺条件 | 10天 | 审核通过或驳回 |

- 关键期最长不超过10天
- 同一时间只能有一个关键期
- 关键期内不会触发新的关键期（事件排队到关键期结束后）
- 关键期体力统一为每天3点，无论第一阶段还是第二阶段
- 关键期内的行动选项由事件类型决定（情境化选择），替换常规季度行动表。例如晋升答辩期间选项为"准备PPT/找领导沟通/打听评委喜好"，而非"埋头工作/学习充电"

### 阶段过渡机制

**从第一阶段进入第二阶段的流程：**

1. **触发条件**：到达 L8 自动触发，或在 L6+ 时通过行动卡片"选择创业"主动触发
2. **过渡选择**（选择卡片弹出）：
   - L8 路径：选择"留任高管"（继续第一阶段，游戏变为公司高管视角）或"辞职创业"（进入第二阶段）
   - L6/L7 创业：直接进入第二阶段
3. **创业启动关键期**（7天）：注册公司、拉合伙人、选办公地点、确定方向。这7天的选择决定初始公司属性
4. **状态继承**：见"第二阶段从第一阶段继承的状态"
5. **NPC 转变**：前同事变为潜在招募对象、投资人、竞争对手或客户。NPC Agent 根据各 NPC 的性格和与玩家的关系决定他们的新角色

## 八、人物属性

所有属性范围 0-100（金钱除外，金钱为具体数字，单位元）。

### 初始值（应届毕业生）

| 属性 | 初始值 | 说明 |
|------|-------|------|
| 健康 | 90 | 年轻人身体好 |
| 专业能力 | 15 | 学校学的和工作需要的有差距 |
| 沟通能力 | 20 | 基本社交能力 |
| 管理能力 | 5 | 几乎为零 |
| 人脉 | 5 | 只认识同学 |
| 心情 | 70 | 对新工作既期待又紧张 |
| 金钱 | 5000元 | 毕业生的全部家当 |
| 声望 | 0 | 谁也不认识你 |

### 属性变化规则

**能力成长（每次行动的收益）：**

| 行动 | 属性变化 |
|------|---------|
| 埋头工作 | 专业能力+3，健康-2 |
| 学习充电 | 选定能力+5 |
| 社交应酬 | 沟通+2，人脉+3，目标NPC好感+5~15（取决于沟通能力），金钱-500 |
| 向上管理 | 直属领导好感+10，随机同事好感-5，沟通+1 |
| 摸鱼休息 | 心情+8，健康+3 |
| 搞副业 | 金钱+2000~8000（随机），20%概率被发现（声望-10，领导好感-20） |
| 跳槽面试 | 人脉+2，可能获得offer（薪资取决于专业能力和声望） |

**健康衰减：**

- 每季度基础衰减：-3
- 住城中村额外：-2/季度
- 进入关键期（高压事件）额外：-1/天
- 恢复途径：摸鱼休息（+3）、运动/健身（+5，需花钱办卡）、体检（花钱，发现问题提前预防）
- 健康≤30：触发"亚健康"状态，所有行动效果减半
- 健康≤10：触发"住院"关键期事件，强制休息，损失1个季度

**心情衰减与恢复：**

- 基础不衰减，由事件驱动（好事+心情，坏事-心情）
- 心情≤30：所有行动效果-30%，社交应酬好感收益减半
- 心情≤10：触发"职业倦怠"事件，可能冲动辞职

**绩效考核（每半年，即每2季度结算一次）：**

计算绩效分数 = (本周期"埋头工作"次数 × 10) + (专业能力 / 2) + 随机波动(-10~+10)

| 绩效分数 | 评级 | 效果 |
|---------|------|------|
| ≥80 | S（卓越） | 加薪15%，优先晋升，声望+5 |
| 60-79 | A（优秀） | 加薪10%，有晋升资格，声望+2 |
| 45-59 | B+（良好） | 加薪5%，有晋升资格 |
| 25-44 | B（合格） | 不加薪 |
| <25 | C（待改进） | 警告，连续2次C触发裁员 |

示例：一个周期内"埋头工作"3次、专业能力50 → 30 + 25 + 随机 = 约55分 → B+

## 九、AI Agent 系统（四个编剧）

### Agent 编排层（调度导演）

编排层是四个 Agent 的总调度，负责协调它们的工作。

**季度结算编排流程：**

```
玩家提交行动分配
  → 游戏引擎：计算硬规则数值变化（能力增减、金钱收支）
  → 世界 Agent：根据当前季度数生成行业环境判定（繁荣/平稳/寒冬）
  → 事件 Agent：根据玩家状态 + 世界环境，决定本季度是否触发事件，输出事件列表
  → 游戏引擎：校验事件合法性（事件不能违反硬规则）
  → NPC Agent：根据玩家行动 + 事件，更新所有 NPC 的状态和反应
  → 叙事 Agent：汇总以上所有输出，生成季度叙事文字
  → 游戏引擎：最终校验所有数值变化，写入存档
```

**关键期编排流程（每日）：**

```
玩家做出当日选择
  → 游戏引擎：计算即时数值变化
  → NPC Agent：生成对话和反应（如晋升答辩时面试官的提问）
  → 事件 Agent：判定是否有突发状况
  → 叙事 Agent：生成当天叙事
  → 游戏引擎：检查关键期结束条件
```

**冲突解决规则：**

- 硬规则永远优先：Agent 输出与硬规则冲突时，硬规则胜出，冲突输出被静默丢弃
- 世界环境优先于事件：事件 Agent 不能在"行业繁荣"时生成"全行业大裁员"
- NPC 一致性校验：编排层检查事件是否与 NPC 性格严重矛盾。若矛盾，该事件被丢弃，事件 Agent 不重新运行（本季度少一个事件，不影响流程）。最多丢弃 2 个事件/季度，防止无限丢弃导致空季度

**Agent 输入输出格式：**

每个 Agent 接收 JSON 格式的上下文，返回 JSON 格式的结构化输出。

**共享输入（游戏状态快照）：**

```
{
  player: { 健康, 专业能力, 沟通, 管理, 人脉, 心情, 金钱, 声望, 职级, 当前住房 },
  actions: [ 本季度玩家选择的行动列表 ],
  npcs: [ { 名字, 身份, 性格摘要, 对玩家好感度, 当前状态 } ],
  company: { 阶段, 行业, 经营状况 },  // 第二阶段额外包含公司财务数据
  history: [ 最近3季度的摘要 ],
  currentQuarter: 数字,
  phase: 1 | 2
}
```

**各 Agent 输出格式：**

| Agent | 输出结构 |
|-------|---------|
| 世界 Agent | `{ economy: "繁荣/平稳/寒冬", trends: [行业趋势文本], companyStatus: "扩张/稳定/收缩", newsItems: [今日条条内容] }` |
| 事件 Agent | `{ events: [{ type, title, description, severity, triggersCritical: bool, durationDays?, statChanges? }], phoneMessages: [麦麦爆料等] }` |
| NPC Agent | `{ npcActions: [{ npcName, action, dialogue?, favorChange, reason }], chatMessages: [小信消息], newNpcs?: [新角色], departedNpcs?: [离场角色] }` |
| 叙事 Agent | `{ narrative: "季度/每日叙事文本", choices?: [关键期选项卡片] }` |

**历史摘要机制：**

每个季度结束时，游戏引擎自动生成该季度的结构化摘要并存入历史：

```
{
  quarter: 数字,
  keyEvents: ["事件A", "事件B"],
  statChanges: { 属性变化差值 },
  npcChanges: ["张伟好感+10", "李雪离职"],
  narrative_summary: "一句话总结"  // 由叙事Agent在季度叙事末尾额外生成
}
```

Agent 输入中只包含最近 3 个季度的摘要，而非完整历史，以控制上下文长度。更早的历史仅在被事件引用时按需加载。

### 1. 故事编剧（叙事 Agent）

**职责：** 把所有发生的事情写成好看的故事。

- 季度总结：将玩家行动 + 事件 + NPC 反应编织成一段有画面感的叙事
- 关键期：每天的细节描写、对话场景、氛围渲染
- 文风：幽默、接地气的打工人口吻，偶尔扎心

### 2. 人物编剧（NPC Agent）

**职责：** 扮演所有 NPC，让每个人有自己的"灵魂"。

- 每个重要 NPC 有独立档案：性格、背景、目标、对玩家的好感度
- 被调用时以该 NPC 视角思考和行动
- 负责对话场景中的 NPC 回应

**初始 NPC：**

| 人物 | 身份 | 性格 | 隐藏目标 |
|------|------|------|---------|
| 王建国 | 直属领导 | 表面和善，实则精于算计 | 想升总监，需要能干的下属出成绩 |
| 张伟 | 同组同事 | 热心但爱八卦 | 也想晋升，视玩家为竞争对手 |
| 李雪 | 隔壁组同事 | 安静内向，技术很强 | 其实在准备跳槽 |
| 赵总 | 部门总监 | 强势、结果导向 | 面临业绩压力，随时可能裁员 |
| 小美 | 前台/行政 | 开朗话多 | 公司八卦情报站 |

NPC 会动态变化：新角色入场（新同事、客户、猎头），老角色离职、调岗、升迁。

**NPC 数量控制：** 同时活跃的重要 NPC 不超过 8 个。当新 NPC 需要入场时，优先让低好感度或低剧情相关性的 NPC 淡出（调岗、离职等）。玩家可在关系面板查看所有历史 NPC，但只有活跃 NPC 会出现在当前叙事中。

**第二阶段 NPC 变化：**

进入第二阶段后，NPC 生态发生根本转变：

| 角色类型 | 来源 | 互动方式 |
|---------|------|---------|
| 合伙人 | 玩家在创业启动关键期招募，可能是前同事 | 高好感度同盟，但可能有分歧 |
| 员工 | 通过"招聘面试"行动获得 | 有满意度，不满会离职 |
| 投资人 | 通过画饼通联系 | 好感决定融资成功率 |
| 竞争对手 | AI 生成或由前同事转变 | 抢市场、挖人 |
| 客户 | 通过"商务拓展"行动获得 | 好感影响续约和营收 |

### 3. 事件编剧（事件 Agent）

**职责：** 制造意外，让游戏不可预测。

每个季度根据当前状态决定是否触发事件：

- **职场事件（高频）：** 加需求、deadline提前、绩效考核、团建、同事离职、办公室政治
- **生活事件（中频）：** 房东涨租、搬家、相亲、家人生病、朋友借钱
- **行业事件（低频高影响）：** 公司被收购、行业裁员潮、竞争对手挖人、公司上市/暴雷

**关键规则：** 参考当前数值状态（健康低容易生病，声望高容易被挖），但保留随机性。某些事件触发"关键期"模式。

**第二阶段事件类型：**

- **经营事件（高频）：** 核心员工要离职、产品出BUG、客户投诉、竞品发布新产品
- **融资事件（中频）：** 投资人约见、尽职调查、估值谈判
- **危机事件（低频高影响）：** 合伙人分歧、大客户流失、现金流断裂、竞争对手恶意挖人、政策变化

### 4. 世界编剧（世界 Agent）

**职责：** 管理大环境，让世界有"呼吸感"。

- **经济周期：** 行业有繁荣和寒冬
- **公司状态：** 公司有经营状况变化，不是一成不变的背景板
- **行业趋势：** 新技术风口、政策变化、竞品动态
- **第二阶段核心：** 决定市场环境，是风口还是寒冬

**涌现机制：** 四个编剧各自独立判断，输出叠加产生不可预测的剧情。世界编剧说"寒冬" → 事件编剧生成"裁员" → 人物编剧让赵总更焦虑 → 故事编剧写出紧张叙事。

## 十一、租房系统

租房是打工人最大的固定开支，直接影响生活质量和游戏策略。

### 房源类型

| 房型 | 月租 | 效果 | 解锁条件 |
|------|------|------|---------|
| 城中村隔断间 | 800 | 心情每季度-5，通勤远（体力上限-1） | 起始可选 |
| 合租单间 | 2000 | 无额外效果，基准选项 | 起始可选 |
| 独立一居室 | 4500 | 心情每季度+3，恢复效果好 | 月薪≥10K |
| 精装两居 | 8000 | 心情+5，可邀请同事来家里（社交加成） | 月薪≥20K |
| 高档公寓 | 15000 | 心情+8，声望+2/季度 | 月薪≥40K |
| 买房(贷款) | 月供20000+ | 心情+10，声望+5/季度，但背上长期负债 | 存款够首付 |

### 策略性

- **省钱住差的** → 存款多，但心情和健康受损
- **花钱住好的** → 心情和社交有加成，但存款增长慢
- **买房** → 稳定感和声望大涨，但被房贷绑死，跳槽和创业要三思

### 租房事件

- 房东突然涨租 / 要卖房让你搬走
- 合租室友奇葩（半夜打游戏、不洗碗）
- 通勤路上偶遇同事/老板
- 小区附近开健身房（花钱办卡恢复健康）
- 房子漏水、停电等生活琐事

### 第二阶段变化

创业后选择公司办公地点（民房办公、孵化器、写字楼、甲级写字楼），影响公司声望和员工满意度。

## 十二、经济系统

### 第一阶段经济

**收入：**

- 月薪按职级固定（见职级表），每季度到账 = 月薪 × 3
- 加薪来源：绩效考核（S/A/B+级别）、晋升、跳槽谈判
- 跳槽溢价：基于声望和专业能力，溢价幅度 10%~50%
- 副业收入：2000~8000/季度（随机），有被发现风险

**固定支出（每季度自动扣除）：**

| 项目 | 金额/季度 | 说明 |
|------|----------|------|
| 房租 | 月租×3 | 取决于当前住房 |
| 餐饮 | 3000 | 基准值，点外卖（饱了吗）额外花费可恢复心情 |
| 交通 | 1500 | 基准值，城中村通勤加500 |
| 日常开销 | 1500 | 生活杂项 |

**可选支出：**

| 项目 | 金额 | 效果 |
|------|------|------|
| 健身卡 | 3000/季度 | 健康+5/季度 |
| 培训课程 | 5000/次 | 选定能力额外+3 |
| 请客吃饭 | 1000/次 | 目标NPC好感+5 |
| 买房首付 | 600000 | 进入房贷状态 |

**存款为负时：** 触发"月光族"状态，心情每季度额外-10。连续2季度为负触发"借贷"事件。

### 第二阶段经济（公司财务）

**公司收入模型：**

- 季度营收 = 基础营收 × 产品质量系数 × 市场环境系数 × (1 + 员工产能加成)
- 基础营收随公司阶段增长：车库期5万、小团队20万、A轮100万、成长期500万、上市冲刺2000万
- 产品质量系数：产品质量/100（范围0.0~1.0）
- 市场环境系数：世界Agent决定（繁荣1.3 / 平稳1.0 / 寒冬0.6）

**公司支出模型：**

- 员工工资 = 员工数 × 平均月薪 × 3
- 办公租金 = 取决于办公地点选择
- 运营成本 = 员工数 × 2000/季度

**公司估值 = 年化营收 × 行业市盈率倍数（AI世界Agent动态调整，范围5~30倍）**

**融资机制：**
- 通过"融资路演"行动触发关键期
- 融资成功率受公司估值、创始人声望、人脉影响
- 融资金额 = 估值 × 出让股份比例（10%~30%）
- 每轮融资稀释创始人股份

## 十三、交互流程

### 一个季度的完整流程

```
开始新季度
  → 阅读上季度叙事总结（故事区）
  → 查看数值变化（仪表盘闪烁提示）
  → 查看手机新消息（小信、麦麦等）
  → 如有事件弹出，先处理事件（可能进入关键期）
  → 分配本季度 10 点体力到各行动
  → 点击"结束本季度"
  → AI 运算，生成下一段叙事
  → 循环
```

### 关键期流程

```
触发关键事件（如晋升答辩）
  → 切换为每日模式（每天 3 点体力）
  → 每天做出具体选择（情境化选项）
  → AI 实时生成当天叙事
  → 事件解决后恢复季度模式
```
