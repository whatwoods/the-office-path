# The Office Path - 打工之道 设计文档

## 一、项目概述

一个基于 AI Agent 工作流的打工模拟器。玩家从应届毕业生入职第一天开始，在规则与 AI 共同编织的开放世界中，努力工作升职加薪。成为公司高管或创立自己的公司后进入第二阶段，从个人成长转向公司运营。

**核心卖点：涌现式剧情** — 多个 AI Agent 独立决策，叠加产生不可预测的剧情走向，连开发者也不知道下一秒会发生什么。

## 二、核心交互形式

**混合模式：**

- **文字叙事** — 主体交互，AI 生成有画面感的故事文字，像读小说一样体验打工人生
- **结构化仪表盘** — 属性面板、关系网、手机面板等可视化组件，让玩家掌握关键信息
- **选择卡片** — 在关键节点弹出行动选项，玩家做出策略决策

## 三、技术栈

| 层面 | 选型 | 理由 |
|------|------|------|
| 框架 | Next.js（全栈） | 前后端一体，AI 代码生成成熟，部署方便 |
| 语言 | TypeScript | 类型安全，AI 编码出错率低 |
| 状态管理 | Zustand | 轻量，适合游戏状态 |
| AI 后端 | 多模型可切换架构 | 抽象 AI 接口层，支持 OpenAI / Claude / DeepSeek 等灵活替换 |
| 数据持久化 | 先 LocalStorage，后续加云存档 | MVP 优先验证核心玩法 |
| UI 风格 | 像素/复古风，中文为主 | 贴合打工人文化调性 |

## 四、系统架构

### 分层架构

整个游戏分为五层，各层职责清晰：

**1. 前端层（玩家看到的）**

三大区域：故事区（文字叙事）、仪表盘（属性/关系/手机）、行动区（选择卡片）。像素复古风。

**2. 游戏引擎层（规则裁判）**

负责硬规则校验。所有数值变更（金钱、属性、职级）必须经过这里。不讲故事，只管算数和判定对错。

**3. Agent 编排层（调度导演）**

每次玩家做出行动，编排层决定调用哪些 Agent、按什么顺序、如何合并它们的输出、如何解决冲突。

**4. Agent 层（四个编剧）**

四个专职 AI Agent，各自独立决策（详见第九节）。

**5. AI 适配层（翻译官）**

统一接口，底层可切换不同 AI 模型。不同 Agent 可以使用不同模型（叙事用强模型，简单判定用便宜模型）。

## 五、游戏阶段

### 第一阶段：打工人成长

玩家从应届毕业生开始，目标是升职加薪，最终成为高管或选择创业。

**职级体系（参照中国互联网大厂）：**

| 职级 | 称呼 | 月薪参考 | 说明 |
|------|------|---------|------|
| L1 | 实习生 | 3K | 起点，还没转正 |
| L2 | 初级员工 | 8K | 转正，干活的螺丝钉 |
| L3 | 中级员工 | 15K | 能独立负责模块 |
| L4 | 高级员工 | 25K | 团队骨干，开始带新人 |
| **L5** | **资深/主管** | **35K** | **分水岭：选技术线还是管理线** |
| L6(技术) | 技术专家 | 50K | 技术线：攻坚难题 |
| L6(管理) | 经理 | 50K | 管理线：带一个小团队 |
| L7(技术) | 高级专家 | 80K | 技术线：架构级影响力 |
| L7(管理) | 总监 | 80K | 管理线：管一个部门 |
| L8 | VP / CTO | 150K+ | 高管层，进入权力中心 |

**晋升条件（硬规则示例）：**

- L2→L3：专业能力≥30，完成过2个项目，在职≥2季度
- L4→L5：专业能力≥60，绩效连续2季度为A/B+，直属领导好感≥50
- L5 选路径：技术线要求专业能力≥70；管理线要求沟通≥50 且 管理能力≥40
- L7→L8：声望≥80，能力多项≥80，需要"机遇"事件触发（如前任离职、公司扩张）

到达 L8 或在 L6+ 时选择辞职创业 → 进入第二阶段。

### 第二阶段：公司运营

不再用个人职级，改为公司发展阶段：

| 阶段 | 说明 |
|------|------|
| 车库创业 | 1-3人，资金紧张，什么都自己干 |
| 小型团队 | 10人以下，开始招人分工 |
| A轮公司 | 拿到融资，30-50人，管理挑战开始 |
| 成长期 | 100+人，部门划分，办公室政治出现 |
| 上市冲刺 | 500+人，对外资本运作，对内组织变革 |
| 上市公司 | 终局，你成了别人故事里的"大厂老板" |

## 六、规则系统（分层设计）

### 硬规则（游戏引擎强制执行）

所有数值计算、晋升条件、经济系统由代码严格控制，AI 不能违反。

### 软规则（AI 自由发挥）

人际关系变化、办公室政治、随机生活事件、行业风云、剧情走向由 AI Agent 自由发挥。

**核心原则：硬规则保证游戏"公平可玩"，软规则保证游戏"有趣意外"。**

## 七、时间与体力系统

### 双节奏时间系统

**平淡期（季度推进）：**

每季度玩家获得 **10 点体力**，分配到各种行动中：

| 行动 | 体力消耗 | 效果 |
|------|---------|------|
| 埋头工作 | 2点 | 提升绩效，小幅提升专业能力 |
| 学习充电 | 2点 | 提升某项能力值（可选方向） |
| 社交应酬 | 2点 | 提升某个NPC好感度或拓展人脉 |
| 向上管理 | 2点 | 提升直属领导好感，但同事可能反感 |
| 摸鱼休息 | 1点 | 恢复心情值 |
| 搞副业 | 3点 | 赚外快，但有被发现的风险 |
| 跳槽面试 | 3点 | 打探外部机会，可能拿到offer |

每季度结算时：

- AI 根据玩家行动分配 + 随机事件，生成季度叙事总结
- 更新数值（工资、能力、关系等）
- 可能触发晋升评审、裁员、跳槽等关键事件

**关键期（按天推进）：**

进入重大事件时（入职第一周、晋升答辩、公司危机、融资谈判等），切换为每天行动，体力按天计算（每天3点），事件结束后恢复季度模式。

## 八、人物属性

| 属性 | 说明 | 影响 |
|------|------|------|
| 健康 | 身体状况，每季度自然下降，加班/高压加速下降 | 太低会生病（强制休息损失行动点），极低触发重大疾病事件 |
| 专业能力 | 核心技能水平 | 绩效、晋升门槛 |
| 沟通能力 | 表达、汇报、谈判 | 向上管理效果、管理线门槛 |
| 管理能力 | 带团队、分任务、决策 | 管理线晋升必须 |
| 人脉 | 行业内的关系网 | 跳槽机会、创业资源 |
| 心情 | 快乐还是emo | 太低影响效率，极低触发离职念头 |
| 金钱 | 存款 | 生存、投资、创业启动资金 |
| 声望 | 公司/行业内的名气 | 高层晋升、创业融资的前提 |

**健康与体力的区别：**

- **体力** — 短期资源，每季度刷新，代表"这个季度有多少精力做事"
- **健康** — 长期指标，不会自动回满，代表"身体还撑得住吗"。通过摸鱼休息、运动、体检等恢复

## 九、AI Agent 系统（四个编剧）

### 1. 故事编剧（叙事 Agent）

**职责：** 把所有发生的事情写成好看的故事。

- 季度总结：将玩家行动 + 事件 + NPC 反应编织成一段有画面感的叙事
- 关键期：每天的细节描写、对话场景、氛围渲染
- 文风：幽默、接地气的打工人口吻，偶尔扎心

### 2. 人物编剧（NPC Agent）

**职责：** 扮演所有 NPC，让每个人有自己的"灵魂"。

- 每个重要 NPC 有独立档案：性格、背景、目标、对玩家的好感度
- 被调用时以该 NPC 视角思考和行动
- 负责对话场景中的 NPC 回应

**初始 NPC：**

| 人物 | 身份 | 性格 | 隐藏目标 |
|------|------|------|---------|
| 王建国 | 直属领导 | 表面和善，实则精于算计 | 想升总监，需要能干的下属出成绩 |
| 张伟 | 同组同事 | 热心但爱八卦 | 也想晋升，视玩家为竞争对手 |
| 李雪 | 隔壁组同事 | 安静内向，技术很强 | 其实在准备跳槽 |
| 赵总 | 部门总监 | 强势、结果导向 | 面临业绩压力，随时可能裁员 |
| 小美 | 前台/行政 | 开朗话多 | 公司八卦情报站 |

NPC 会动态变化：新角色入场（新同事、客户、猎头），老角色离职、调岗、升迁。

### 3. 事件编剧（事件 Agent）

**职责：** 制造意外，让游戏不可预测。

每个季度根据当前状态决定是否触发事件：

- **职场事件（高频）：** 加需求、deadline提前、绩效考核、团建、同事离职、办公室政治
- **生活事件（中频）：** 房东涨租、搬家、相亲、家人生病、朋友借钱
- **行业事件（低频高影响）：** 公司被收购、行业裁员潮、竞争对手挖人、公司上市/暴雷

**关键规则：** 参考当前数值状态（健康低容易生病，声望高容易被挖），但保留随机性。某些事件触发"关键期"模式。

### 4. 世界编剧（世界 Agent）

**职责：** 管理大环境，让世界有"呼吸感"。

- **经济周期：** 行业有繁荣和寒冬
- **公司状态：** 公司有经营状况变化，不是一成不变的背景板
- **行业趋势：** 新技术风口、政策变化、竞品动态
- **第二阶段核心：** 决定市场环境，是风口还是寒冬

**涌现机制：** 四个编剧各自独立判断，输出叠加产生不可预测的剧情。世界编剧说"寒冬" → 事件编剧生成"裁员" → 人物编剧让赵总更焦虑 → 故事编剧写出紧张叙事。

## 十一、租房系统

租房是打工人最大的固定开支，直接影响生活质量和游戏策略。

### 房源类型

| 房型 | 月租 | 效果 | 解锁条件 |
|------|------|------|---------|
| 城中村隔断间 | 800 | 心情每季度-5，通勤远（体力上限-1） | 起始可选 |
| 合租单间 | 2000 | 无额外效果，基准选项 | 起始可选 |
| 独立一居室 | 4500 | 心情每季度+3，恢复效果好 | 月薪≥10K |
| 精装两居 | 8000 | 心情+5，可邀请同事来家里（社交加成） | 月薪≥20K |
| 高档公寓 | 15000 | 心情+8，声望+2/季度 | 月薪≥40K |
| 买房(贷款) | 月供20000+ | 心情+10，声望+5/季度，但背上长期负债 | 存款够首付 |

### 策略性

- **省钱住差的** → 存款多，但心情和健康受损
- **花钱住好的** → 心情和社交有加成，但存款增长慢
- **买房** → 稳定感和声望大涨，但被房贷绑死，跳槽和创业要三思

### 租房事件

- 房东突然涨租 / 要卖房让你搬走
- 合租室友奇葩（半夜打游戏、不洗碗）
- 通勤路上偶遇同事/老板
- 小区附近开健身房（花钱办卡恢复健康）
- 房子漏水、停电等生活琐事

### 第二阶段变化

创业后选择公司办公地点（民房办公、孵化器、写字楼、甲级写字楼），影响公司声望和员工满意度。

## 十二、经济系统

- 每季度发薪（月薪×3），扣固定生活开支（房租、吃饭、交通）
- 绩效考核每半年一次，参照大厂的 361/271 强制分布
- 加薪随晋升或年度调薪，跳槽可谈溢价
- 第二阶段：公司有季度财报（收入、支出、现金流、估值）

## 十三、交互流程

### 一个季度的完整流程

```
开始新季度
  → 阅读上季度叙事总结（故事区）
  → 查看数值变化（仪表盘闪烁提示）
  → 查看手机新消息（小信、麦麦等）
  → 如有事件弹出，先处理事件（可能进入关键期）
  → 分配本季度 10 点体力到各行动
  → 点击"结束本季度"
  → AI 运算，生成下一段叙事
  → 循环
```

### 关键期流程

```
触发关键事件（如晋升答辩）
  → 切换为每日模式（每天 3 点体力）
  → 每天做出具体选择（情境化选项）
  → AI 实时生成当天叙事
  → 事件解决后恢复季度模式
```
